Why choose the MMS

The Mix Messaging Server (MMS) is a great option to chat with your friends if you don't want to use social media applications such as Discord, Instagram, or Facebook.

This is because you have 100% control over MMS. You can read your friends' messages with ease, as they are stored in a single file. And MMS is super private. MMS does not have most of the built-in privacy features that you see today with famous social-media networks such as Twitter or Reddit, but it gives you the opportunity to set those up yourselves. With MMS, you don't have to worry about others spying on your messages. You can talk about literally anything, and don't have to worry about anyone looking at those messages.

The recommended way to set up MMS is using the Tor Relay, for which you can find many guides on the internet. Otherwise, you can simply port-forward your server on some proxy, or ask one of your friends to setup a proxy for you, which you can connect to, sacrificing their safety, and chat about whatever sick secrets you want to with your friends >:D

Do note, networking tends to get pretty tedious and technical at times. A better guide for setting up MMS on the Tor relay will come soon!

Configuration

When you run the binary and generate the configuration file, it gives you an option to edit the file. If you do not choose to edit right away, it prints the path to the YAML configuration file. It is located at start/config. To edit the configuration file, use your preferred editor, e.g. nano. nano start/config/pref.yaml The contents should look like this (provided you did not edit beforehand): 

# Edit the PORT value to choose a custom port
# However, you can restart the client and edit it via that
                
# Edit the ADDR value to choose a custom address to start the server on
# You can also restart the client and edit the ADDR via that
                
# Example ports: 22, 80, 443, 3000, 5000, 8000
# Example addrs: 127.0.0.1, localhost, 192.168.69.42
                
PORT: foo
ADDR: bar
                
# View the documentation @ https://mick.gdn/wiki/mms for more info

Ports & Addresses

You might be confused by the 'PORT' and 'ADDR' fields. This section should explain those in detail. Let's take a closer look: 

PORT: foo
ADDR: bar
Of course. Using the popular 'foo' and 'bar' to represent unknown variables, we have two fields. PORT and ADDR. Anyone who has been on the internet for long enough knows what ports and addresses are, but maybe not in detail.

Please read this text about ports, and this text about addresses. A port is needed to port-forward a web service running locally. This application runs locally. When you start the Flask server, it hosts itself on whatever port you specified alongside the IP address. But how is that of any significance on localhost?

'127.0.0.1' or more commonly known as 'localhost' is the address only your computer, and SOLELY your own computer can access. Even if you are on a LAN with other devices that can interact with your computer, they won't be able to access your computer's localhost (127.0.0.1). They will be able to access your computer by its reserved address.

Let's say you host the application on localhost (127.0.0.1) on another computer, but you have a port-forwarding app like PacketRiot. You SSH into the computer and start the PKTRIOT app on your computer. It won't be able to find any web services running. This is because the Flask server is running on that computer's localhost. You need to start the Flask app on the computer's private address. (e.g. 192.168.18.49).

I won't be teaching how to port forward, but this is almost everything you need to know to get it up and running.

Using PacketRiot

Believe it or not, it's pretty easy. Just make sure you use the right port and IP address. Make an account on PacketRiot, which is explained in detail over here. After you are done, execute the following command: 

$ pktriot http YOUR_PORT
Make sure to run it on the computer your Flask server is running. You also have the option to manually port-forward the server, but that will be explained in another guide later.
